Overview

Our first Advanced cohort was for students who already had the fundamentals and wanted to write software that lasts. The focus shifted from "make it work" to "make it good": how real systems are structured, how experienced developers keep code readable as it grows, and how to carry a larger project from idea to finished result.

What students explored

  • Software architecture — how to structure a program so it can grow without collapsing.
  • Design patterns — proven solutions to problems that recur in every codebase.
  • Clean code & refactoring — naming, structure and improving code without breaking it.
  • Data structures — choosing the right tool for the job.
  • Working in a larger codebase — navigating and extending code at realistic scale.
  • Real-world project work — building something substantial, end to end.

Skills developed

  • Architectural thinking — designing before diving in.
  • Writing code other people (and future-you) can read and maintain.
  • Confidence to take on projects well beyond toy programs.
